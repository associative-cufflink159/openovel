Why openovel

Fast reading loop, durable story memory

Foreground narrator

A latency-sensitive model call focuses on prose and the next meaningful beat.

Background agents

Resident agents update world state, pacing, cards, memory, and guidance after turns.

File-native stories

Canon, notes, context, state, and memory are inspectable files, not a hidden database.

How it works

A dual-loop runtime for long-running fiction

The foreground loop gets the reader prose quickly. The background loop records consequences and prepares better context for future turns.

Features

Built for playable stories that keep going

  • Desktop story library with create, rename, import, restart, delete, and export.
  • Suggested choices that remain optional; readers can type anything free-form.
  • Reading-paced reveal independent of provider stream speed.
  • Optional rich rendering, scene backdrops, image generation, TTS, and comic mode.
  • Settings for built-in and OpenAI-compatible providers.
